## PV Provisioners
[deployment.yaml](../practice/5.saving-data/3.pvc/deployment.yaml)

Мы выяснили, что использование дисков может быть неэффективным. Например диски будут нарезаны по 10Gb, 
а claim будут по 10Mb. Они займут все PV, несмотря на то, что они по 10Gb. 

Так же, нам неудобно ждать админа, который придет и будет руками нарезать нам диски, да еще непонятных рамеров. Если все 
PV будут заняты, либа не будет PV подходящих под запрашиваемые параметры, наши поды будут висеть в статусах 
Pending и ждать, пока появятся ресурсы.

![12.1 Provisioner.png](img/12.1%20Provisioner.png)

Для этого придумали Provisioners. Это программа, которая умеет сама ходит в облако и нарезать там диск, 
причем по заказанному размеру. Получать там данные о дисках и пилить PV

Накатим PVC
```bash
kubectl apply -f pvc.yaml
```

Накатим CM
```bash
kubectl apply -f configmap.yaml  
```

Накатим Deployment
```bash
kubectl apply -f deployment.yaml 
```

Получим поды и посмотрим
```bash
NAME                         READY   STATUS    RESTARTS   AGE
fileshare-69786c8b5c-v2fbg   1/1     Running   0          6s
fileshare-69786c8b5c-x6hx2   1/1     Running   0          6s
```
